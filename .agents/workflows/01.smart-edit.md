🛠️ Smart Edit Workflow

**Reference:** `ARCHITECTURE.md` § Development Workflow, `/linting-gates`

---

## 📋 Quy trình 3 bước

### 1️⃣ Locate → Plan
- `grep -n "pattern" file.js` → find line
- `Read file.js (offset: line, limit: 30)` → verify context & indentation
- Plan: Edit vs Write? (Edit for existing files, Write for new files only)

### 2️⃣ Execute
- **Edit (existing)**: old_string phải khớp 100% (whitespace, indentation, newlines)
- **Write (new)**: Use for brand new files only
- Copy old_string exactly from Read output (không thêm/bớt space)

### 3️⃣ Verify
- `Read file.js (offset: line, limit: 30)` → confirm changes applied
- `npm run lint` → **0 errors AND 0 warnings** (both must pass)
- Check context around change is intact

---

## 🛠️ Edit vs Write

| Situation | Tool |
|-----------|------|
| Fix typo/bug in existing file | ✅ Edit |
| Update variable, replace logic | ✅ Edit |
| Multiple edits in same file | ✅ Edit (multi-replace) |
| Create brand new file | ✅ Write |
| Change existing file completely | ❌ **Never** (use Edit) |

**Critical:** Write on existing file = lose entire file. Always use Edit.

**Pattern 1: Token replacement**

## ⚠️ Mandatory Rules

1. **Minimal diffs** — only change what needs changing
2. **100% exact match** — whitespace, indentation, newlines must match exactly
3. **Never Write existing files** — always use Edit (Write deletes entire file)
4. **Verify after edit** — read back file and check syntax
5. **Zero-violation linting** — `npm run lint` must show **0 errors AND 0 warnings** (both must be fixed)

---

## ⚡ Quick Errors & Fixes

| Issue | Cause | Fix |
|-------|-------|-----|
| Edit fails, "not found" | old_string doesn't match exactly | grep to find exact text, copy with whitespace |
| Overwrites entire file | Used Write instead of Edit | Use Edit for existing files only |
| Lint shows warnings | Syntax issue or style violation | Fix all warnings, not just errors |
| Linting fails | Broke brackets/quotes or tokens wrong | Reread file, fix issue, lint again |
| Changes in wrong place | old_string not unique | Add more context lines around change |

**Critical:** Linting warnings are NOT acceptable. Fix ALL violations (errors + warnings).

---

## 🔴 Zero-Violation Linting Gate

After every edit, linting must show:
```
✅ 0 errors
✅ 0 warnings
```

If linting shows ANY violations (errors OR warnings):
1. Stop — don't consider edit complete
2. Fix the violation in source code
3. Re-run `npm run lint`
4. Repeat until 0 errors AND 0 warnings

**Example:**
```
npm run lint
→ 1 error, 2 warnings
→ UNACCEPTABLE — fix all 3 violations
→ npm run lint again
→ 0 errors, 0 warnings
→ ✅ DONE
```
