✅ Linting Gates Workflow

Workflow này quy định cách chạy, chẩn đoán, và khắc phục các lỗi linting CSS/JavaScript trước khi commit. Đây là cổng bắt buộc (mandatory gate) — không được commit nếu có lỗi hoặc warning.

**Reference:** `ARCHITECTURE.md` § Development Workflow, `.stylelintrc.json`, `eslint.config.mjs`

---

## 📋 Quy trình xác minh Linting

### Bước 1: Chạy Linting Toàn Bộ (Run Full Lint)

```bash
npm run lint
```

**Output format:**

```
✓ CSS Linting completed (0 warnings, 0 errors)
✓ JavaScript Linting completed (0 problems)

✓ All linting gates passed
```

Nếu có lỗi → khu vực tiếp theo sẽ hiển thị chi tiết.

### Bước 2: Chẩn đoán CSS Linting Errors (CSS Diagnostics)

Nếu `npm run lint` báo lỗi CSS, chạy:

```bash
npm run lint:css
```

**Đầu ra ví dụ:**

```
renderer/css/design-system/atoms/button.css
 42:5   ✖ Unexpected invalid hex color "#1234GG"   color-no-invalid-hex

renderer/css/styles.css
 105:1  ✖ Unexpected duplicate selector ".my-class"   no-duplicate-selectors
 205:8  ✖ Unexpected unspaced operator in calc()     function-calc-no-unspaced-operator
```

**Danh sách CSS linting rules:**

| Rule | Lỗi | Fix |
|------|-----|-----|
| `color-no-invalid-hex` | Hex color sai format (#1234GG, #GGGGGG) | Dùng hex valid: #000 đến #FFF hoặc #000000 đến #FFFFFF |
| `no-duplicate-selectors` | Selector được định nghĩa 2 lần | Gộp 2 block vào 1 definition |
| `length-zero-no-unit` | `0px`, `0em` — zero không cần unit | Thay thành `0` (không unit) |
| `function-calc-no-unspaced-operator` | `max(0,calc(...))` không space | Thêm space: `max(0, calc(...))` |
| `import-notation` | `@import url(...)` | Thay thành `@import "...";` |

**Ví dụ sửa:**

**Lỗi:** Hex color invalid
```css
/* WRONG */
--ds-accent: #1234GG;

/* CORRECT */
--ds-accent: #ff9800;  /* 6 hex digits hoặc 3 hex digits */
```

**Lỗi:** Duplicate selector
```css
/* WRONG */
.btn { padding: 8px; }
.btn { color: red; }

/* CORRECT */
.btn {
  padding: 8px;
  color: red;
}
```

**Lỗi:** Unspaced operator in calc()
```css
/* WRONG */
--ds-radius-inset: max(0,calc(var(--ds-radius) - 8px));

/* CORRECT */
--ds-radius-inset: max(0, calc(var(--ds-radius) - 8px));
```

**Lỗi:** Unit on zero
```css
/* WRONG */
margin: 0px;
padding: 0em;

/* CORRECT */
margin: 0;
padding: 0;
```

### Bước 3: Chẩn đoán JavaScript Linting Errors (JS Diagnostics)

Nếu `npm run lint` báo lỗi JavaScript, chạy:

```bash
npm run lint:js
```

**Đầu ra ví dụ:**

```
renderer/js/core/app.js
  42:10  error   'unusedVar' is assigned a value but never used   no-unused-vars
  55:8   error   'x == y' is unsafe, use '===' instead             eqeqeq
  78:3   error   'console.log' is not allowed                      no-console

renderer/js/modules/editor.js
  120:15  error   'undefinedModule' is not defined                 no-undef
```

**Danh sách JavaScript linting rules:**

| Rule | Lỗi | Fix |
|------|-----|-----|
| `no-unused-vars` | Biến/tham số khai báo nhưng không dùng | Prefix với `_`: `_unused`, `_index` hoặc xóa nếu thực sự không cần |
| `eqeqeq` | `==` hoặc `!=` (loose equality) | Dùng `===` hoặc `!==` |
| `no-console` | `console.log(...)` | Dùng `console.warn()` hoặc `console.error()` thay thế |
| `no-undef` | Sử dụng biến không được định nghĩa | Thêm `window.[ModuleName] = [ModuleName];` ở cuối file |
| `no-var` | Khai báo `var` | Dùng `let` hoặc `const` thay thế |

**Ví dụ sửa:**

**Lỗi:** Unused variable
```javascript
/* WRONG */
const currentIndex = 5;
console.log('Done');  // currentIndex không dùng

/* CORRECT */
const _currentIndex = 5;  // Prefix _ cho ESLint
console.log('Done');

// Hoặc xóa hoàn toàn nếu không cần
console.log('Done');
```

**Lỗi:** Loose equality
```javascript
/* WRONG */
if (state == null) { ... }
if (value != undefined) { ... }

/* CORRECT */
if (state === null) { ... }
if (value !== undefined) { ... }
```

**Lỗi:** console.log không được phép
```javascript
/* WRONG */
console.log('Debug:', myValue);

/* CORRECT - nếu cần log cho development */
console.warn('Debug:', myValue);
console.error('Critical:', myValue);

/* Hoặc xóa hoàn toàn nếu không cần */
```

**Lỗi:** Missing window.* export
```javascript
/* WRONG */
const MyModule = (() => {
  return { init: () => { ... } };
})();
// Quên: window.MyModule = MyModule;

/* CORRECT */
const MyModule = (() => {
  return { init: () => { ... } };
})();
window.MyModule = MyModule;  // ← Explicit export
```

**Lỗi:** Unused catch parameter
```javascript
/* WRONG */
try {
  someAsync();
} catch (error) {
  console.warn('Fallback');  // error không dùng
}

/* CORRECT */
try {
  someAsync();
} catch (_error) {  // Prefix _ cho ESLint
  console.warn('Fallback');
}
```

### Bước 4: Sửa lỗi (Fix Errors)

**Cách tiếp cận:**

1. **Để ESLint tự sửa (khi có thể):**
   ```bash
   npm run lint:js -- --fix
   ```
   
   Các lỗi này sẽ tự sửa:
   - `no-var` → tự đổi `var` thành `let`
   - `eqeqeq` → tự đổi `==` thành `===`
   - Một số warning khác

2. **Sửa thủ công (khi cần context):**
   - `no-unused-vars` → xóa hoặc prefix `_` tùy ngữ cảnh
   - `no-console` → thay `console.log` với `warn`/`error` hoặc xóa
   - `no-undef` → thêm `window.*` export hoặc kiểm tra script order
   - Lỗi CSS → sửa cấu trúc CSS

### Bước 5: Xác minh sau sửa (Verify After Fix)

Chạy linting lại để đảm bảo hết lỗi:

```bash
npm run lint
```

**Kết quả kỳ vọng:**
```
✓ CSS Linting completed (0 warnings, 0 errors)
✓ JavaScript Linting completed (0 problems)

✓ All linting gates passed
```

Nếu vẫn có lỗi → quay lại Bước 2 hoặc 3, xác định lỗi cụ thể.

## 🛡️ Cổng Commit (Commit Gate)

**BẮTBUỘC trước khi commit:**

```bash
# 1. Chạy full linting
npm run lint

# 2. Kiểm tra output — PHẢI có "0 errors, 0 problems"
# Nếu có error/warning → TUYỆT ĐỐI KHÔNG COMMIT

# 3. Nếu hết lỗi → có thể commit
git add .
git commit -m "..."
```

**Nguyên tắc:**
- ❌ Không commit nếu `npm run lint` báo error hoặc warning
- ✅ Chỉ commit khi "All linting gates passed"
- ⚠️ Nếu cấu trúc quá lớn, hãy commit module riêng (không gom vào 1 commit khổng lồ)

```javascript
import pluginJs from "@eslint/js";

  {

        ...globals.browser,
        AppState: "readonly",

```

## ⚠️ Nguyên tắc quan trọng

1. **Zero Tolerance Policy**
   - Linting **PHẢI** 0 errors, 0 warnings
   - Không được ignore rules hoặc comment ngoại lệ
   - Nếu rule không phù hợp → thảo luận với team, update config chung

2. **Fix Ngay Lập Tức**
   - Phát hiện lỗi → sửa ngay, không để tích tụ
   - Lỗi tích tụ khó fix, lỗi sớm dễ sửa

3. **Commit Gate bắt buộc**
   - Không bao giờ bypass linting checks
   - Nếu gặp tình huống đặc biệt → thảo luận trước, không self-approve

4. **Version Updates**
   - Khi cập nhật `stylelint` hoặc `eslint` → có thể có lỗi mới
   - Luôn chạy `npm run lint` sau khi update dependencies
   - Fix lỗi mới trước khi tiếp tục phát triển

5. **Documentation + Testing**
   - Khi thêm rule mới → update documentation này
   - Khi disable rule → ghi lý do trong comment
   - Luôn test linting setup trên file mới trước deploy
