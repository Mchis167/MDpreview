📋 Artifact Documentation Workflow

Workflow này quy định khi nào sử dụng Artifact, cấu trúc chuẩn, và cách cập nhật để giữ chat box sạch sẽ và tài liệu có tổ chức.

**Reference:** `/plan`, `/changelog`, `/console-test`

---

## 📋 Quy trình Artifact vs Chat

### Sử dụng Chat Box (Ngắn gọn)

- ✅ Xác nhận yêu cầu ("Hiểu rồi, tôi sẽ...")
- ✅ Thông báo hoàn thành ("Đã cập nhật X file...")
- ✅ Câu hỏi làm rõ ("Bạn muốn dùng Token X hay Y?")
- ✅ Tóm tắt ngắn (1-2 câu chính)

**Độ dài:** Max 5 câu, chỉ quan trọng nhất

### Sử dụng Artifact (Chi tiết)

- ✅ **Plan/Design**: Implementation plan, architecture decisions
- ✅ **Test Cases**: Test suite, verification checklist
- ✅ **Audit Report**: Code analysis, refactor assessment
- ✅ **Technical Docs**: Hướng dẫn chi tiết, troubleshooting
- ✅ **Changelog**: Version history (khi được yêu cầu)

**Độ dài:** > 500 words hoặc cần cấu trúc phức tạp

## 🛠️ Quy trình tạo Artifact

### Bước 1: Xác định loại Artifact (Identify Type)

| Loại | Tên file | Nội dung | Ví dụ |
|------|----------|---------|-------|
| Plan | `implementation_plan.md` | Implementation steps, files to change, risks | "Refactor tabs module" |
| Test Suite | `test_cases.md` | Functional/UI/Integration tests | "Editor module tests" |
| Audit | `audit_report.md` | Code analysis, improvements | "Design system audit" |
| Design | `design_proposal.md` | Architecture decisions, trade-offs | "Token migration strategy" |
| Guide | `user_guide.md` | Step-by-step instructions | "How to add new token" |

### Bước 2: Tạo cấu trúc chuẩn (Standard Structure)

```markdown
---
title: [Tiêu đề rõ ràng]
created: 2026-04-25
updated: 2026-04-25
status: [draft | in-progress | completed]
---

# [Tiêu đề]

## 🎯 Mục tiêu

[1-2 dòng: Mục đích của tài liệu này]

## 📋 Nội dung chính

[Chi tiết, sử dụng bảng/danh sách/code blocks]

## ⚠️ Lưu ý / Rủi ro

[Các vấn đề cần chú ý]

## ✅ Kiểm tra / Hoàn thành

[Checklist hoặc tiêu chí xác minh]

---

**Last updated:** 2026-04-25
```

### Bước 3: Tóm tắt trong Chat

Gửi tóm tắt **ngắn gọn** trong chat:

```
Tôi đã tạo artifact [Implementation Plan for Tabs Refactor].

📌 Điểm chính:
- 3 file sẽ thay đổi (tabs.js, tab-bar.js, app.js)
- Estimated time: 2 giờ
- Risk: Kiểm tra scroll position kỹ lưỡng

👉 Chi tiết đầy đủ: [Artifact link]
```

**Không copy nội dung artifact vào chat.**

### Bước 4: Cập nhật khi có thay đổi (Maintain)

**Thay vì:**
```
Actually, I changed my mind. Instead of X, we should do Y.
Let me explain...
[10 dòng giải thích]
```

**Làm:** Cập nhật trực tiếp trong artifact + notify trong chat:

```
✏️ Đã cập nhật Implementation Plan: 
- Changed approach from Y to X (cleaner)
- Updated file changes list
- New checklist added

👉 Chi tiết: [Artifact link]
```

## 📏 Độ dài và Cấu trúc

### Plan/Design Documents

```
Title
↓
Objective (1 paragraph)
↓
Key Points (2-5 sections)
↓
Files to Change (table/list)
↓
Step-by-Step (numbered list)
↓
Risks / Considerations
↓
Verification Checklist
```

**Độ dài:** 1,000-2,000 words (phù hợp để review)

### Test Cases

```
Feature Overview
↓
Test Categories (Functional | UI/UX | Integration | Edge Cases)
↓
Test Table (ID | Description | Steps | Expected | Status)
↓
Console Helpers (optional)
↓
Notes / Known Issues
```

**Độ dài:** 500-1,500 words

### Audit Reports

```
Executive Summary
↓
Findings (Issues categorized: Critical | Major | Minor)
↓
Detailed Analysis (per finding)
↓
Recommendations (prioritized)
↓
Implementation Roadmap
↓
Verification Steps
```

**Độ dài:** 1,500-3,000 words

## ⚠️ Nguyên tắc quan trọng

1. **Ngôn ngữ: Tiếng Việt**
   - Tất cả artifact phải viết bằng Tiếng Việt
   - Code blocks giữ nguyên (JavaScript, CSS, bash)

2. **Chat + Artifact Separation**
   - Chat: Ngắn gọn, chỉ điều cần biết ngay
   - Artifact: Chi tiết, context đầy đủ
   - Tỷ lệ: 80% artifact, 20% chat

3. **Cập nhật liên tục**
   - Artifact không bao giờ "cũ"
   - Khi kế hoạch thay đổi → cập nhật artifact ngay
   - Giữ trạng thái (draft/in-progress/completed)

4. **Metadata rõ ràng**
   - `created`, `updated`, `status` bắt buộc
   - `author` (nếu collaboration)
   - `version` (nếu có nhiều revisions)

5. **Tham khảo các workflow khác**
   - `/plan` → tạo implementation_plan artifact
   - `/changelog` → tham khảo cấu trúc changelog
   - `/console-test` → test cases artifact

## 🧪 Verification Checklist

- [ ] Artifact có metadata (title, created, updated, status)
- [ ] Cấu trúc rõ ràng (Objective → Content → Checklist)
- [ ] Viết bằng Tiếng Việt (code blocks ngoài lệ)
- [ ] Tóm tắt < 5 câu trong chat
- [ ] Link tới artifact được cung cấp
- [ ] Khi cập nhật → notify trong chat
