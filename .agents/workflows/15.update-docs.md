📄 Update Docs Workflow

Workflow này kiểm tra xem các thay đổi trong session có ảnh hưởng đến function docs không, và chỉ cập nhật khi thực sự cần thiết.

**Docs cần maintain:** `docs/function-docs/`

---

## 📋 Quy trình

### 1️⃣ Scan thay đổi của session

Xác định các file đã bị sửa trong session:

```bash
git diff --name-only HEAD
```

Hoặc đọc lại lịch sử hội thoại để liệt kê các file đã Edit/Write.

---

### 2️⃣ Map file → doc liên quan

So sánh file thay đổi với bảng dưới đây:

| File thay đổi | Doc cần review |

| `renderer/js/modules/tabs.js` | `docs/function-docs/TABS.md` |

---

### 3️⃣ Đánh giá "có cần update không?"

Với mỗi doc liên quan, đọc doc hiện tại và so sánh với code mới:

**Cần update nếu:**
- ✅ Function signature thay đổi (tên, params, return)
- ✅ Behavior của function thay đổi đáng kể
- ✅ Function mới được thêm vào module
- ✅ Function bị xóa / deprecated
- ✅ Flow hoạt động thay đổi (ví dụ: thêm bước mới)
- ✅ Keyboard shortcut thay đổi
- ✅ localStorage key thay đổi

**Không cần update nếu:**
- ❌ Chỉ fix typo hoặc comment trong code
- ❌ Refactor nội bộ, behavior không đổi
- ❌ CSS/style thay đổi không ảnh hưởng logic
- ❌ Chỉ thêm/sửa log message

---

### 4️⃣ Thực hiện update (nếu cần)

> [!CAUTION]
> **TUYỆT ĐỐI KHÔNG dùng `Write` trên file doc đã tồn tại.** `Write` sẽ xóa toàn bộ nội dung cũ. Chỉ dùng `Edit`.

**Bắt buộc trước khi edit:**
1. `Read` toàn bộ file doc hiện tại
2. Xác định **chính xác** đoạn text cần thay đổi (copy nguyên từ Read output)
3. Chọn loại thao tác phù hợp (xem bảng bên dưới)

---

**Bảng quyết định thao tác:**

| Tình huống | Thao tác | Tool |
|---|---|---|
| Sửa behavior của function đã có | Replace đúng đoạn mô tả | `Edit` (old_string → new_string) |
| Sửa signature / params | Replace đúng dòng header `### \`fn()\`` | `Edit` |
| Thêm function MỚI vào module | **Append** vào cuối section liên quan | `Edit` (thêm vào sau function cuối cùng của nhóm) |
| Thêm tính năng mới (không phải function) | **Append** vào cuối section phù hợp | `Edit` |
| Xóa function đã bị remove | Replace block function đó bằng rỗng | `Edit` |
| Đánh dấu deprecated | Chỉ sửa dòng header, giữ nguyên nội dung | `Edit` |

---

**Ví dụ — Sửa behavior (Edit surgical):**
```
old_string: "Debounced 500ms — POST toàn bộ state lên server."
new_string: "Debounced 300ms — POST toàn bộ state lên server. Bỏ qua nếu không có thay đổi."
```

**Ví dụ — Thêm function mới (Append vào đúng chỗ):**
```
old_string: (đoạn kết thúc của function liền trước, ví dụ:)
"### `getActive()`\nTrả về workspace đang active.\n\n---"

new_string: (giữ nguyên old, thêm block mới vào sau)
"### `getActive()`\nTrả về workspace đang active.\n\n### `getAll()`\nTrả về danh sách tất cả workspaces.\n\n---"
```

**Ví dụ — Deprecated:**
```
old_string: "### `oldFunction()`"
new_string: "### ~~`oldFunction()`~~ *(deprecated — dùng `newFunction()` thay thế)*"
```

**Thêm doc mới cho module chưa có:**
Dùng `Write` (chỉ cho file **mới hoàn toàn chưa tồn tại**) — tham khảo cấu trúc của `TABS.md` hoặc `EDITOR.md`.

## ⚠️ Nguyên tắc quan trọng

> [!IMPORTANT]
> **IFF NEED** — Chỉ cập nhật doc khi có thay đổi thực sự ảnh hưởng đến behavior hoặc API. Đừng update doc chỉ vì "cho chắc".

> [!CAUTION]
> **KHÔNG BAO GIỜ dùng `Write` trên file doc đã tồn tại** — `Write` xóa toàn bộ file, mất hết nội dung cũ.
> Chỉ dùng `Write` khi tạo file doc **hoàn toàn mới** (chưa tồn tại).

- **Read trước khi Edit** — Luôn đọc file doc trước, copy chính xác `old_string` từ output
- **Surgical edits** — Chỉ thay đổi đúng đoạn liên quan, giữ nguyên toàn bộ phần còn lại
- **Append cho tính năng mới** — Thêm vào cuối section, không chen ngang giữa các entry cũ
- **Ngôn ngữ nhất quán** — Giữ ngôn ngữ (Tiếng Việt / English) như doc hiện tại
- **Timestamp** — Cập nhật dòng `*Document — YYYY-MM-DD*` ở cuối file
- **Không xóa thông tin cũ** trừ khi function đó đã bị remove hoàn toàn khỏi code
