🧪 Test Case Canvas Workflow

Workflow này quy định cách tạo Test Case Artifact toàn diện, bao gồm functional tests, UI/UX tests, integration tests, và edge cases.

**Reference:** `ARCHITECTURE.md` § Development Workflow, `/artifact-docs`, `/console-test`

---

## 📋 Quy trình thực hiện

### Bước 1: Phân tích tính năng (Analyze Feature)

**Hiểu rõ:**
- **User story**: Người dùng muốn làm gì?
- **Main flow**: Happy path (bình thường nhất)
- **Alternate flows**: Các biến thể khác
- **Error handling**: Cái gì có thể sai?
- **Edge cases**: Những trường hợp biên
- **Dependencies**: Module/service khác được sử dụng
- **State**: AppState hoặc local state thay đổi?

**Ví dụ - Feature: File Save**
```
User story: User clicks Save → file persisted to disk
Main flow: Edit → Save → success notification
Alternate: Save with no changes → "nothing to save" message
Error: Disk full → error message
Edge case: Rapid saves (debounce?)
Dependencies: FileService, AppState
State: AppState.current.isDirty
```

### Bước 2: Thiết kế Test Suite (Design Test Suite)

Chia test cases theo **4 danh mục**:

#### A. Functional Tests (Core logic)

Cái gì phải hoạt động bình thường:

| ID | Description | Steps | Expected | Status |
|----|-------------|-------|----------|--------|
| F1 | Happy path | 1. Do X → 2. Verify Y | Y is true | ☐ |
| F2 | Return value correct | 1. Call function(arg) | Returns expected value | ☐ |
| F3 | State updated | 1. Trigger action | AppState.x = new value | ☐ |

#### B. UI/UX Tests (Visual + interaction)

Cái gì về giao diện:

| ID | Description | Steps | Expected | Status |
|----|-------------|-------|----------|--------|
| U1 | Button visible | 1. Load page | Button rendered | ☐ |
| U2 | Click handling | 1. Click button | Handler fires | ☐ |
| U3 | Visual feedback | 1. Hover button | Hover state visible | ☐ |
| U4 | Accessibility | 1. Tab to button | Focus outline visible | ☐ |
| U5 | Responsive | 1. Desktop 1920px | Layout correct | ☐ |
| U5b | Responsive | 2. Mobile 375px | Layout correct | ☐ |

#### C. Integration Tests (Module interactions)

Cái gì tương tác với module khác:

| ID | Description | Steps | Expected | Status |
|----|-------------|-------|----------|--------|
| I1 | FileService call | 1. Trigger save | FileService.save called | ☐ |
| I2 | AppState update | 1. Action → 2. Check state | State updated correctly | ☐ |
| I3 | Sidebar reflect change | 1. Do action → 2. Check sidebar | Sidebar updates | ☐ |

#### D. Edge Cases & Error Handling

Cái gì có thể sai:

| ID | Description | Steps | Expected | Status |
|----|-------------|-------|----------|--------|
| E1 | Null/undefined input | 1. Call function(null) | Graceful handling (no crash) | ☐ |
| E2 | Empty state | 1. With 0 items | Message: "No items" | ☐ |
| E3 | Large dataset | 1. With 10K items | Performance acceptable | ☐ |
| E4 | Network error | 1. Fetch fails | Error message shown | ☐ |
| E5 | Rapid clicks | 1. Click button 10x fast | Debounced (single execution) | ☐ |
| E6 | Missing permission | 1. User no permission | Permission error shown | ☐ |

### Bước 4: Console Helpers (Optional)

Nếu feature khó test thủ công, tạo console scripts:

```javascript
// Helper: Setup test environment
(async function setupTest() {
  console.group('🧪 Test Setup: [Feature]');
  
  // Arrange
  const mockData = { /* ... */ };
  await AppState.loadFile('test-file.md');
  
  // Act
  console.log('✅ Test environment ready');
  console.log('AppState:', AppState.current);
  
  console.groupEnd();
})();

// Helper: Verify behavior
(async function verifyFeature() {
  console.group('🔍 Verify: [Feature]');
  
  const result = await FeatureModule.doSomething();
  console.log('✅ Result:', result);
  console.log('Expected: [X]');
  console.log('Match:', result === expected ? '✅' : '❌');
  
  console.groupEnd();
})();
```

Chèn vào artifact để user có thể copy-paste vào console.

### Bước 5: Gửi Artifact và Chờ Approval

**Chat message (ngắn gọn):**

```
🧪 Tôi đã tạo Test Case Suite cho [Feature Name].

Bao gồm:
- Functional (5 tests)
- UI/UX (6 tests)
- Integration (3 tests)
- Edge cases (6 tests)
- Console helpers (for manual testing)

👉 Chi tiết: [Artifact link]

Ready to start testing?
```

**Chờ user:**
- ✅ Approval để bắt đầu test
- 🔄 Điều chỉnh test cases
- ❓ Câu hỏi về feature

### Bước 6: Execute Tests & Verify

**Khi user ready:**

1. **Run console helpers** (nếu có)
2. **Manual test** theo từng test case:
   - Follow steps
   - Check expected result
   - Mark pass/fail
3. **Automated tests** (nếu có):
   - Unit tests: `npm test`
   - E2E tests: Browser automation
4. **Update artifact** với status (✅/❌)

## 📋 Test Case Template (Standard)

```markdown
| ID | Description | Steps | Expected | Status |
|----|-------------|-------|----------|--------|
| F1 | [1-line test name] | 1. [Step 1]<br/>2. [Step 2]<br/>3. [Step 3] | [Expected outcome] | ☐ |
```

**Quy tắc:**

- **Description**: Rõ ràng, test cái gì?
- **Steps**: Numbered, actionable, clear
- **Expected**: Cụ thể, có thể verify (không mơ hồ)
- **Status**: ☐ (pending) → ✅ (pass) → ❌ (fail)

---

## ⚠️ Nguyên tắc quan trọng

1. **Toàn diện (Comprehensive)**
   - Bao gồm cả happy path + edge cases
   - Không chỉ test main flow mà cả error handling
   - ~20-30 test cases per feature là lý tưởng

2. **Cụ thể (Specific)**
   - Test case phải dễ hiểu, dễ thực hiện
   - Steps rõ ràng, không mơ hồ
   - Expected result phải có thể verify

3. **Coverage (Bao phủ)**
   - Functional: Tất cả logic paths
   - UI/UX: Tất cả interactions + responsive sizes
   - Integration: Tất cả module dependencies
   - Edge: Boundary conditions + error cases

4. **Realistic (Thực tế)**
   - Test như user thực tế sử dụng
   - Không test implementation details
   - Focus on observable behavior

5. **Maintainable (Bảo trì)**
   - Test cases phải lâu dài (update khi feature thay đổi)
   - Không hardcode magic numbers
   - Tham khảo ARCHITECTURE.md cho constraints

## 🧪 Test Levels

| Level | Where | Example |
|-------|-------|---------|
| **Unit** | Single function | `FileService.save(file)` returns Promise |
| **Integration** | Module → Module | FileService calls AppState.update() |
| **E2E** | Full flow | User saves file → sidebar updates → notification shown |
| **Manual** | Browser | Visual check, responsive, interactions |

---

## 📊 Test Case Statistics

**Healthy test suite:**

- Functional: 30-40% of tests
- UI/UX: 20-30%
- Integration: 20-30%
- Edge cases: 10-20%

**Example (Feature with 20 tests):**
- 6-8 functional
- 4-6 UI/UX
- 4-6 integration
- 2-4 edge cases

## 🧪 Verification Checklist

- [ ] Test cases cover happy path
- [ ] Test cases cover edge cases + errors
- [ ] Test cases cover UI/UX interactions
- [ ] Test cases cover integration points

- [ ] All expected results are specific and verifiable

- [ ] Total ~20-30 tests for medium-sized feature
