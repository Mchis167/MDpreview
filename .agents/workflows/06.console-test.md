🧪 Console Automation Test Workflow

Workflow này quy định cách AI Agent tạo ra các đoạn mã JavaScript để kiểm thử tự động (Unit Test/Integration Test) trực tiếp trên Console của trình duyệt.

**Reference:** `ARCHITECTURE.md` § JavaScript Module System, § Development Workflow

## 📋 Quy trình thực hiện

1.  **Xác định phạm vi (Scope)**:
    *   Xác định rõ hàm, module hoặc tính năng UI nào cần kiểm thử.
    *   Liệt kê các điều kiện tiền đề (ví dụ: phải đang mở file, phải ở Edit Mode).

2.  **Thiết kế Script**:
    *   Sử dụng cấu trúc **AAA** (Arrange - Act - Assert).
    *   Bọc toàn bộ script trong một IIFE (Immediately Invoked Function Expression) hoặc `async function` để tránh xung đột biến toàn cục.

3.  **Trình bày kết quả**:
    *   Sử dụng `console.group` để phân cấp các hạng mục test.
    *   Sử dụng icon trạng thái: ✅ (Pass), ❌ (Fail), ⚠️ (Warning), 📄 (Info).

## 🛠️ Cấu trúc Script Chuẩn

```javascript
/**
 * [Tên Tính Năng] - Automation Test
 */
(async function runTest() {
    console.group('🧪 Testing: [Mô tả tính năng]');
    
    // 1. Setup (Chuẩn bị)
    // Kiểm tra các biến toàn cục (AppState, EditorModule, v.v.)
    
    // 2. Execution (Thực thi)
    // Giả lập click, nhập liệu hoặc gọi hàm API
    
    // 3. Assertion (Đối soát)
    // So sánh kết quả thực tế với kỳ vọng
    
    console.groupEnd();
})();
```

## ⚠️ Quy tắc an toàn
1.  **Không phá hủy dữ liệu**: Hạn chế gọi các lệnh xóa (Delete) mà không có cảnh báo.
2.  **Khôi phục trạng thái**: Nếu script thay đổi dữ liệu để test, hãy cố gắng khôi phục lại trạng thái ban đầu sau khi kết thúc.
3.  **Xử lý bất đồng bộ**: Luôn sử dụng `await` khi gọi các hàm trả về Promise (như `save`, `fetch`).
