📝 Plan-First Workflow

Workflow này bắt buộc AI phải phân tích kỹ lưỡng và lập kế hoạch thực hiện (Implementation Plan) trước khi bắt đầu viết code thật.

## 📋 Quy trình thực hiện

### Bước 1: Phân tích yêu cầu (Analyze)

- **Đọc yêu cầu**: Hiểu rõ mục tiêu, scope, constraints
- **Khám phá codebase**: 
  - `find`, `grep` để xác định files liên quan
  - `read` để xem nội dung quan trọng
  - Hiểu dependencies và interactions
- **Xác định impact**: Các file/module sẽ bị ảnh hưởng

### Bước 2: Tạo Implementation Plan Artifact

**Sử dụng `/artifact-docs` workflow**

Kế hoạch phải bao gồm:

- **Objective** (1 paragraph): Mục đích chính
- **Scope**: Cái gì được include, cái gì không
- **Files to Change** (table):
  - File path
  - Reason for change
  - Type of change (add/modify/delete)
- **Step-by-Step** (numbered list):
  - Sequential steps
  - Order matters (dependencies)
  - Estimated effort per step
- **Risks & Considerations**:
  - Architectural constraints (tokens, IIFE pattern, etc.)
  - Potential side effects
  - Testing considerations
- **Verification Checklist**:
  - How to verify each step
  - Success criteria
  - Linting gates required

**Ví dụ structure:**
```markdown
# Implementation Plan: [Feature Name]

## 🎯 Objective
[1 paragraph: what we're building and why]

## 📋 Files to Change
| File | Change | Reason |
|------|--------|--------|
| renderer/js/modules/editor.js | Add init logic | Bootstrap new feature |
| renderer/css/design-system/atoms/button.css | Update token usage | Consistency |

## 🛠️ Step-by-Step
1. Update CSS tokens in design-system/tokens.css
2. Modify button.css to use new tokens
3. Add initialization in editor.js
4. Test in browser

## ⚠️ Risks
- Ensure linting passes after CSS changes
- Verify all component variants still work

## ✅ Verification
- npm run lint → 0 errors
- Visual test in browser
```

### Bước 3: Chờ xác nhận (Wait for Approval)

**Dừng và chờ feedback:**

```
📋 Tôi đã tạo Implementation Plan tại [link].

Điểm chính:
- 3 files sẽ thay đổi
- ~2 giờ estimated
- Cần verify CSS tokens cẩn thận

👉 Bạn có đồng ý không? Cần điều chỉnh gì không?
```

**BẮTBUỘC:**
- ❌ Không sửa code
- ❌ Không commit
- ❌ Không push
- Chỉ chờ user approval

### Bước 4: Thực thi (Execute)

Sau khi user approve:
- Sử dụng `/smart-edit` hoặc `Edit` tool
- Theo đúng step-by-step plan
- Chạy `npm run lint` sau mỗi file change
- Update plan nếu có deviation

## ⚠️ Nguyên tắc quan trọng
*   Luôn tạo artifact kế hoạch trước.
*   Tuyệt đối không viết code "nháp" hoặc code "thật" vào project khi chưa có kế hoạch được phê duyệt.
*   Nếu kế hoạch thay đổi trong quá trình làm, hãy cập nhật lại artifact kế hoạch.
