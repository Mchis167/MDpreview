✂️ Refactor to Atomic Design Workflow

Workflow này quy định cách phân tích, bóc tách, và chuyển đổi các component cũ (legacy) thành các component chuẩn trong Design System.

**Reference:** `ARCHITECTURE.md` § CSS Architecture, § Component Hierarchy, `/create-component`, `/standardize-ds`

---

## 📥 Input Requirements

Trước khi bắt đầu refactor, xác định:

- **Target**: Khối HTML cần refactor (ID hoặc class selector)
- **New Name**: Tên component mới (kebab-case)
- **Target Level**: `atoms`, `molecules`, hoặc `organisms`
- **Current Location**: File chứa code cũ

---

## 🛠️ Quy trình thực hiện

### Bước 1: Phân tích & Thu thập (Audit Legacy Code)

**1.1 Đọc HTML hiện tại:**
```bash
# Tìm HTML element
grep -n "id=\"[target]\"" renderer/index.html
grep -n "class=\".*[target].*\"" renderer/index.html
```

**1.2 Tìm CSS liên quan:**
```bash
# Tìm CSS selector
grep -rn "\.[target]\|#[target]" renderer/css/
```

**1.3 Tìm JavaScript logic:**
```bash
# Tìm event listeners hoặc manipulation
grep -rn "[target]" renderer/js/
grep -rn "querySelector.*[target]\|getElementById.*[target]" renderer/js/
```

**1.4 Tài liệu kiến trúc hiện tại:**
- Hiểu cấu trúc HTML
- Xác định CSS properties được sử dụng
- Xác định event listeners và logic
- Xác định state management (AppState hoặc local)

### Bước 2: Tạo bản kế hoạch (Create Refactor Plan)

**Sử dụng artifact (yêu cầu approval):**

Plan phải bao gồm:

- **Inventory** (mục lục):
  - HTML structure (bao nhiêu elements)
  - CSS properties used (hardcode values?)
  - JavaScript logic (interactive?)
  - State dependencies

- **Mapping** (ánh xạ):
  ```
  Old HTML (selector/class) → New component structure
  Old CSS file → New CSS file (with tokens)
  Old JS (app.js) → New JS module (IIFE)
  Old state (AppState.x) → New state (or same path)
  ```

- **Conversion Plan**:
  - Step 1: Create new CSS file (replace hardcodes with tokens)
  - Step 2: Create new JS module (IIFE pattern)
  - Step 3: Move HTML template / create render function
  - Step 4: Register in design-system.css + index.html
  - Step 5: Migrate state (if needed)
  - Step 6: Clean up old code

- **Risks**:
  - Will visual appearance match exactly?
  - Are there edge cases in behavior?
  - Are there other code paths using old selectors?

**Chờ user approval trước khi proceed.**

### Bước 3: Tạo Component mới (Create New Component)

**Sử dụng `/create-component` workflow:**

1. **Create CSS file** (`renderer/css/design-system/<level>/<name>.css`):
   - Convert all hardcoded colors/spacing → CSS tokens
   - Keep layout + behavior exactly the same
   - Add tokens from tokens.css (no hardcodes!)

   ```css
   /* Old CSS */
   .old-component {
     background: #ffffff;
     color: #333333;
     padding: 12px 16px;
     border-radius: 8px;
   }

   /* New CSS */
   .ds-[name] {
     background: var(--ds-bg-base);
     color: var(--ds-text-primary);
     padding: var(--ds-space-md) var(--ds-space-lg);
     border-radius: var(--ds-radius-md);
   }
   ```

2. **Create JS module** (if interactive):
   - IIFE pattern (mandatory)
   - Move event listeners from old code
   - Keep same behavior + state management
   - Add to window.* exports

   ```javascript
   const [NewName] = (() => {
     const SELECTORS = {
       root: '.ds-[name]',
       // migrate old selectors
     };

     function _initElement(element) {
       // Move old event listeners here
       element.addEventListener('click', _handleClick);
     }

     function _handleClick(event) {
       // Old logic from app.js or old event handler
     }

     return { init: () => { /* ... */ } };
   })();
   window.[NewName] = [NewName];
   ```

### Bước 4: Di chuyển Logic (Migrate Logic)

**4.1 HTML Template Migration:**

Nếu HTML cũ được hardcode trong index.html:
- Giữ lại `<div class="ds-[name]-root"></div>` (mount point)
- Hoặc generate HTML trong JS render() function

**4.2 Event Listeners Migration:**

```javascript
// Old code (trong app.js hoặc sự kiện inline)
document.getElementById('old-btn').addEventListener('click', () => {
  AppState.updateSomething();
});

// New code (trong JS module)
function _initElement(element) {
  element.querySelector('.ds-[name]-btn').addEventListener('click', () => {
    AppState.updateSomething();  // Same state interaction
  });
}
```

**4.3 State Management:**

Nếu component sử dụng AppState:
- Giữ nguyên AppState references
- Không tạo new state ngoài AppState
- Update AppState listeners nếu cần

### Bước 5: Đăng ký Component mới (Register)

**Sử dụng `/create-component` step 3-5:**

1. Add CSS import to `design-system.css`
2. Add script to `index.html` (correct order)
3. Add init call to `app.js`
4. Run `npm run lint` → must be 0 errors

### Bước 6: Xóa Code cũ (Cleanup Legacy)

**6.1 Xóa HTML cũ từ index.html:**
```html
<!-- REMOVE THIS SECTION -->
<div id="old-component" class="old-style">
  ...old HTML...
</div>

<!-- REPLACE WITH MOUNT POINT (if needed) -->
<div class="ds-[name]-root"></div>
```

**6.2 Xóa CSS cũ:**
```bash
# Tìm và xóa selectors từ CSS files
grep -rn "\.old-class\|#old-id" renderer/css/

# Xóa từ layout.css, modals.css, styles.css, etc.
```

**6.3 Xóa logic cũ từ JavaScript:**
- Xóa event listeners cũ từ app.js
- Xóa helper functions cũ
- Xóa state management cũ (nếu không được sử dụng ở đâu khác)

**6.4 Search and verify:**
```bash
# Ensure old selectors are gone
grep -rn "old-selector\|old-class\|old-id" renderer/

# Should return 0 results
```

### Bước 7: Xác minh (Verify & Test)

**7.1 Linting:**
```bash
npm run lint
# Must be: 0 errors, 0 warnings
```

**7.2 Visual Verification:**
- Component visual = cũ 100%?
- Interactions work exactly same?
- No console errors?
- Responsive? (if applicable)

**7.3 Browser Console Test:**
```javascript
// Verify component exported
console.log(typeof window.[NewName]);  // Should be 'object'
console.log(window.[NewName].init);    // Should be function

// Verify old selectors gone
console.log(document.querySelector('.old-selector'));  // Should be null
```

**7.4 Verify other usages:**
```bash
# Search codebase for references to old component
grep -rn "old-component\|old-selector" renderer/
# Should return 0 results (except in this workflow doc)
```

### Bước 8: Cập nhật Documentation (Document)

**File:** `ARCHITECTURE.md`

```markdown
## Refactoring: [Old Component] → [New Component]

**Date**: 2026-04-25
**Status**: Completed

**Old Structure**:
- HTML: Hardcoded in index.html
- CSS: layout.css, styles.css

**New Structure**:

- JS: renderer/js/components/[level]/[name].js
- Export: window.[NewName]

**Migration Notes**:
- All hardcodes → tokens
- Event listeners migrated
- State management: AppState (unchanged)
```

## ⚠️ Nguyên tắc quan trọng

1. **Visual Parity First**
   - New component phải trông giống y hệt component cũ
   - Không thay đổi UI/UX trong refactor
   - Behavior phải giống 100%

2. **Tokens Mandatory**
   - Không bao giờ hardcode màu/spacing trong CSS mới
   - ❌ `color: #333;`
   - ✅ `color: var(--ds-text-primary);`

3. **Complete Cleanup**
   - Xóa TOÀN BỘ code cũ sau refactor
   - Không giữ lại "backup" hoặc duplicate
   - Grep để verify: old selectors phải = 0 results

4. **Linting Gates**
   - `npm run lint` phải 0 errors trước commit
   - Không bao giờ bypass linting

5. **Test Thoroughly**
   - Không chỉ hình ảnh mà interaction cũng phải test
   - Edge cases: empty states, long text, etc.
   - Responsive: desktop, tablet, mobile (nếu applicable)

## 🔍 Common Pitfalls

| Problem | Solution |
|---------|----------|
| Hardcoded colors/sizes | Replace ALL with tokens |
| Old CSS not deleted | Use grep to find and remove |
| Event listeners duplicated | Remove old listeners, keep only in new module |
| Linting fails | Fix before proceeding |
| Visual mismatch | Compare pixel-by-pixel, adjust new CSS |
