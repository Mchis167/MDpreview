🏗️ Module Creation Workflow

Workflow này quy định cách tạo mới các JavaScript module tuân thủ kiến trúc IIFE (Immediately Invoked Function Expression) với explicit window.* exports, kèm theo quy trình tích hợp vào index.html và xác minh linting.

**Reference:** `ARCHITECTURE.md` § JavaScript Module System, § Component Hierarchy

---

## 📋 Quy trình tạo Module

### Bước 1: Xác định loại Module (Identify Module Type)

Trả lời câu hỏi này để biết module thuộc danh mục nào:

- **Atom Component**: Đơn vị UI nhỏ nhất (Button, Input, Switch, Toggle) — Không dependency vào component khác
  - File path: `renderer/js/components/atoms/[name].js`
  - Export: `window.[NameComponent]`
  - Ví dụ: `window.IconActionButton`, `window.SwitchToggle`

- **Molecule Component**: Nhóm Atom (SearchComponent, ContextMenu, SidebarSectionHeader) — Có thể depend vào Atoms
  - File path: `renderer/js/components/molecules/[name].js`
  - Export: `window.[NameComponent]`
  - Ví dụ: `window.ContextMenu`, `window.SearchComponent`

- **Organism Component**: Đơn vị UI lớn (RightSidebar, TabBar, TreeView) — Có thể depend vào Atoms + Molecules
  - File path: `renderer/js/components/organisms/[name].js`
  - Export: `window.[NameComponent]`
  - Ví dụ: `window.RightSidebar`, `window.TabBar`

- **Service Module**: Logic dùng chung (FileService, MarkdownLogicService, TreeDragManager) — Không render UI
  - File path: `renderer/js/services/[name]-service.js`
  - Export: `window.[NameService]` hoặc `window.[Name]` tùy convention
  - Ví dụ: `window.FileService`, `window.MarkdownLogicService`

- **Module (Controller)**: Quản lý tính năng lớn (TabsModule, EditorModule, TreeModule) — Tổng hợp các component + service
  - File path: `renderer/js/modules/[name].js`
  - Export: `window.[Name]Module`
  - Ví dụ: `window.TabsModule`, `window.EditorModule`

- **Utility**: Hàm tiện ích (zoom.js, scroll.js, code-blocks.js) — Pure functions, không state
  - File path: `renderer/js/utils/[name].js`
  - Export: `window.[NameUtils]` hoặc `window.[Name]Util`
  - Ví dụ: `window.ZoomUtil`, `window.CodeBlocksUtil`

### Bước 2: Tạo cấu trúc Module chuẩn (Create Module Structure)

**Cấu trúc file bắt buộc:**

```javascript
/**
 * [Module Name] — [Short description]
 * Purpose: [What problem does this solve]
 * Dependencies: [List modules/services this depends on, if any]
 */
const [ModuleName] = (() => {
  'use strict';

  // ============================================
  // Private State
  // ============================================
  let state = {
    // private variables
  };

  // ============================================
  // Private Functions
  // ============================================
  function _helperFunction() {
    // implementation
  }

  function _anotherHelper() {
    // implementation
  }

  // ============================================
  // Public API
  // ============================================
  return {
    init: function() {
      // initialization logic
    },
    publicMethod: function(param) {
      // public interface
    },
    // ... other public methods
  };
})();

// Explicit export to global scope
window.[ModuleName] = [ModuleName];
```

**Quy tắc:**
- Tất cả private functions bắt đầu với `_` (underscore) để tránh ESLint no-unused-vars warning
- State được khai báo trong private scope, không bao giờ trực tiếp trên window
- Public API được return từ IIFE — rõ ràng xác định các method/property có thể gọi ngoài
- Sử dụng 'use strict' để bắt lỗi hữu ích

### Bước 3: Khai báo Dependencies (Declare Dependencies)

Nếu module sử dụng component hoặc service khác, **phải khai báo tại đầu file:**

```javascript
/**
 * My Custom Component
 * Dependencies: IconActionButton, FileService, TabsModule
 */
```

**Quy tắc phụ thuộc:**
- Atom → không phụ thuộc vào bất kỳ component nào
- Molecule → chỉ phụ thuộc vào Atom
- Organism → có thể phụ thuộc vào Atom + Molecule
- Service → không phụ thuộc vào component
- Module → có thể phụ thuộc vào Component + Service

**Kiểm tra phụ thuộc:**

```bash
# Tìm nơi sử dụng service trong file
grep -n "window.FileService\|FileService\." renderer/js/components/molecules/my-component.js
```

Nếu tìm thấy → đúng là có phụ thuộc.

### Bước 4: Tích hợp vào index.html (Register in Script Pipeline)

**File:** `renderer/index.html`

**Locate thứ tự tải script chính xác:**

Script pipeline order (BẮTBUỘC):
1. `renderer/js/core/` — Core infrastructure (app.js, electron-bridge.js) — **Phải load đầu tiên**
2. `renderer/js/design-system/` — Design system (nếu có)
3. **Components (Atoms → Molecules → Organisms)** — Đúng thứ tự dependency
4. `renderer/js/services/` — Services (không phụ thuộc UI)
5. `renderer/js/utils/` — Utilities (pure functions)
6. `renderer/js/modules/` — Modules (tổng hợp)

**Ví dụ cấu trúc:**

```html
<!-- CORE (MUST BE FIRST) -->
<script src="js/core/electron-bridge.js" defer></script>

<!-- DESIGN SYSTEM COMPONENTS -->
<script src="js/components/atoms/switch-toggle.js" defer></script>
<script src="js/components/atoms/textarea.js" defer></script>

<!-- MOLECULES (depend on atoms) -->
<script src="js/components/molecules/context-menu.js" defer></script>

<!-- ORGANISMS (depend on atoms + molecules) -->
<script src="js/components/organisms/tab-bar.js" defer></script>

<!-- SERVICES -->
<script src="js/services/file-service.js" defer></script>

<!-- MODULES (depend on everything) -->
<script src="js/modules/editor.js" defer></script>

<!-- ENTRY POINT (MUST BE LAST) -->
<script src="js/core/app.js" defer></script>
```

**Quy tắc:**
- Mỗi script có `defer` attribute
- Core modules phải load **trước** bất kỳ thứ gì khác
- Hãy tuân thủ thứ tự: Atoms → Molecules → Organisms → Services → Utilities → Modules → App
- Entry point (app.js) **phải load cuối cùng**

### Bước 5: Kiểm tra linting (Verify Linting)

Chạy linting để đảm bảo module không vi phạm quy chuẩn mã:

```bash
npm run lint
```

**Các lỗi phổ biến khi tạo module:**

1. **no-unused-vars (Unused parameters)**
   - ❌ `function handleEvent(event, ignoredParam) { ... }`
   - ✅ `function handleEvent(event, _ignoredParam) { ... }`

2. **no-undef (Missing window.* export)**
   - ❌ Quên thêm `window.MyModule = MyModule;` ở cuối file
   - ✅ Luôn thêm export statement

3. **eqeqeq (Loose equality)**
   - ❌ `if (state == null) { ... }`
   - ✅ `if (state === null) { ... }`

Không được commit nếu lint fail.

### Bước 6: Cập nhật ARCHITECTURE.md (Update Documentation)

**File:** `ARCHITECTURE.md` § JavaScript Module System

Thêm module vào danh sách phù hợp:

```markdown
## Module: [Module Name]
- **Path**: `renderer/js/[category]/[name].js`
- **Type**: [Atom/Molecule/Organism/Service/Module/Utility]
- **Purpose**: [1-line description]
- **Public API**:
  - `init()` — Initialize module
  - `publicMethod(param)` — Description
- **Dependencies**: [List if any]
```

Ví dụ:

```markdown
## Module: IconActionButton (Atom Component)
- **Path**: `renderer/js/components/atoms/icon-action-button.js`
- **Type**: Atom Component
- **Purpose**: Render compact icon-only action buttons (24x24 or 32x32)
- **Public API**:
  - `render(config)` — Render button and return DOM element
  - `destroy()` — Clean up resources
- **Dependencies**: None
```

## 🧪 Testing Module Integration

Sau khi tạo module, verify nó hoạt động bằng console test:

```javascript
// In browser console
(async function testModule() {
  console.group('🧪 Testing: [Module Name]');
  
  // 1. Check if module is exported to window
  console.log('✅ Module exported:', typeof window.[ModuleName] !== 'undefined');
  
  // 2. Check if public API exists
  console.log('✅ Public API available:', window.[ModuleName].publicMethod !== undefined);
  
  // 3. Call init if available
  if (window.[ModuleName].init) {
    window.[ModuleName].init();
    console.log('✅ Initialization successful');
  }
  
  console.groupEnd();
})();
```

## ⚠️ Nguyên tắc quan trọng

1. **IIFE Pattern bắt buộc**
   - Luôn bọc code trong IIFE để tránh global scope pollution
   - Private state và function không bao giờ rò rỉ ra ngoài

2. **Explicit exports only**
   - Không sử dụng ẩn sự phụ thuộc giữa module
   - `window.[ModuleName] = [ModuleName];` **bắt buộc**

3. **Dependencies phải có thứ tự load đúng**
   - Script phải load TRƯỚC khi được sử dụng
   - Nếu MyModule dùng FileService, FileService phải load trước

4. **Naming convention đạo hàm từ loại module**
   - Component: `[Name]Component` hoặc `[Name]` (ví dụ: `IconActionButton`, `ContextMenu`)
   - Service: `[Name]Service` hoặc `[Name]` (ví dụ: `FileService`, `MarkdownLogicService`)
   - Module: `[Name]Module` (ví dụ: `EditorModule`, `TabsModule`)
   - Utility: `[Name]Util` hoặc `[Name]Utils` (ví dụ: `ZoomUtil`, `CodeBlocksUtil`)

5. **Không hardcode dependency vào window**
   - ❌ `const service = window.FileService;` ở top level (phụ thuộc vào thứ tự load)
   - ✅ `const service = window.FileService;` bên trong function (lazy lookup)

6. **Testing và verify bắt buộc**
   - Chạy `npm run lint` TRƯỚC commit
   - Chạy console test để verify module exports + API
   - Không commit nếu có lỗi hoặc warning
