🎨 Token Management Workflow

Workflow này quy định cách thêm, cập nhật, hoặc xóa Design Tokens trong hệ thống 3-tier (Primitives → Alpha Palette → Semantic).

**Reference:** `ARCHITECTURE.md` § Token System (Design Tokens)

---

## 📋 Quy trình thêm Token mới

### Bước 1: Xác định tầng (Identify Tier)

Trả lời câu hỏi này để biết token thuộc tầng nào:

- **Tier 1 (Primitives)**: Giá trị nguyên tố được dùng nhiều nơi (màu mới, spacing mới, radius mới)
  - Ví dụ: `--ds-primitive-blue: #2196f3`
  
- **Tier 2 (Alpha Palette)**: Opacity variant của một màu (chưa có trong palette)
  - Ví dụ: `--ds-blue-a10: rgba(33, 150, 243, 0.10)`
  
- **Tier 3 (Semantic)**: Mục đích sử dụng được đặt tên (backgrounds, text, interactive)
  - Ví dụ: `--ds-accent-blue: var(--ds-primitive-blue)`

### Bước 2: Kiểm tra tồn tại (Check Existence)

Trước khi thêm, kiểm tra xem token đã tồn tại hay chưa:

```bash
grep -n "YOUR_TOKEN_NAME" renderer/css/design-system/tokens.css
```

Nếu đã tồn tại → **Update** (bước 3b), không cần tạo mới.

### Bước 3a: Thêm Tier 1 hoặc Tier 2

**File:** `renderer/css/design-system/tokens.css`

Locate vị trí đúng trong file:
- Tier 1 (Primitives) — Khoảng line 5–60
- Tier 2 (Alpha Palette) — Khoảng line 70–150

Ví dụ thêm Tier 1:
```css
/* Colors */
--ds-primitive-orange: #ffbf48;
--ds-primitive-base: #151515;
--ds-primitive-blue: #2196f3;    /* ← NEW */
```

Ví dụ thêm Tier 2:
```css
/* Blue with opacity */
--ds-blue-a10: rgba(33, 150, 243, 0.10);    /* ← NEW */
--ds-blue-a15: rgba(33, 150, 243, 0.15);
--ds-blue-a20: rgba(33, 150, 243, 0.20);
```

### Bước 3b: Thêm Tier 3 (Semantic)

**File:** `renderer/css/design-system/tokens.css`

Locate section **Tier 3 — Semantic** (khoảng line 150+)

Phân loại vào nhóm đúng:
- `/* Backgrounds */`
- `/* Text Colors */`
- `/* Interactive */`
- `/* Status */`
- `/* Semantic Radius */`
- `/* Shadows */`
- `/* Transitions */`

Ví dụ thêm vào "Interactive":
```css
/* Interactive */
--ds-accent: var(--ds-primitive-orange);
--ds-accent-hover: #ffaa00;
--ds-accent-green: #4cb150;
--ds-accent-blue: #2196f3;    /* ← NEW */
```

### Bước 4: Verify Linting

Chạy linting để đảm bảo token được định nghĩa đúng:

```bash
npm run lint:css
```

Nếu có lỗi → Fix trước, không commit nếu có error.

### Bước 5: Cập nhật ARCHITECTURE.md

Nếu thêm Tier 3 token mới (ngữ nghĩa), thêm nó vào tài liệu:

**File:** `ARCHITECTURE.md` § Token System (Tier 3)

Thêm dòng này vào ví dụ Tier 3 tương ứng:
```css
--ds-accent-blue: #2196f3;
```

## 📝 Quy trình cập nhật Token hiện có

### Trường hợp 1: Thay đổi giá trị Tier 1

Nếu cần thay đổi màu hoặc spacing:

```css
/* OLD */
--ds-primitive-orange: #ffbf48;

/* NEW */
--ds-primitive-orange: #ff9800;
```

**⚠️ Cảnh báo:** Thay đổi Tier 1 ảnh hưởng đến **TẤT CẢ** Tier 2 và Tier 3 dùng nó.
- Verify toàn bộ app UI sau khi thay đổi.
- Commit message phải rõ lý do thay đổi.

### Trường hợp 2: Thêm opacity variant thiếu

Nếu component cần `--ds-white-a35` nhưng chỉ có `a30` và `a40`:

Thêm vào Tier 2 (Alpha Palette):
```css
--ds-white-a35: rgba(255, 255, 255, 0.35);
```

Chạy linting, update ARCHITECTURE.md, commit.

### Trường hợp 3: Rename Tier 3 token

Nếu `--ds-text-muted` không clear lắm, đổi thành `--ds-text-tertiary`:

**Quy trình:**
1. Tạo token mới: `--ds-text-tertiary: var(--ds-text-muted);`
2. Find-replace toàn codebase: `--ds-text-muted` → `--ds-text-tertiary`
3. Xóa token cũ từ `tokens.css`
4. Run `npm run lint` để confirm
5. Commit với message "refactor: rename text-muted → text-tertiary"

---

## ⚠️ Nguyên tắc quan trọng

1. **Tier 1 dùng Hex, Tier 2 dùng rgba()**
   - Tier 1: `#ffbf48` (primitive)
   - Tier 2: `rgba(255, 191, 72, 0.10)` (with RGB from Tier 1)
   - Tier 3: `var(--ds-...)` (reference tầng dưới)

2. **Luôn dùng `var()` cho referencing**
   - ✓ `--ds-accent-blue: var(--ds-primitive-blue);`
   - ✗ `--ds-accent-blue: #2196f3;` (hardcode trong Tier 3 là lỗi)

3. **Naming convention**
   ```
   --ds-[category]-[value]-[variant]
   
   ✓ --ds-text-primary
   ✓ --ds-white-a05
   ✓ --ds-shadow-md
   ✗ --ds-muted-color (không rõ)
   ```

4. **Documentation is code**
   - Token mới → Update ARCHITECTURE.md
   - Không update docs → Cấm commit

5. **Run linting EVERY time**
   ```bash
   npm run lint
   ```
   Không được commit nếu lint fail.

## 🧪 Testing Token Usage

Sau khi thêm token, verify nó được dùng đúng:

```javascript
// In browser console
const styles = getComputedStyle(document.documentElement);
const value = styles.getPropertyValue('--ds-accent-blue').trim();
console.log('Token value:', value);  // Should output: #2196f3
```
